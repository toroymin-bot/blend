## 🐛 BUG-005 — localStorage QuotaExceededError 미처리

### 증상
대용량 대화/문서가 누적되어 origin 별 localStorage quota(보통 5–10 MB)에 도달하면 `setItem`이 `QuotaExceededError`를 throw. 일부 스토어는 try-catch가 있었지만 대부분 `console.warn`만 찍혀 **사용자가 저장이 실패한 사실을 알 수 없었음**.

### 픽스

#### 1. `src/lib/safe-storage.ts` (신규)
```ts
export type StorageQuotaDetail = { store: string; key: string; approxBytes: number };

export function safeSetItem(key: string, value: string, store = 'unknown'): boolean {
  try { localStorage.setItem(key, value); return true; }
  catch (err) {
    if (isQuotaError(err)) {
      console.warn(`[safe-storage] quota exceeded writing "${key}" (store=${store}, ~${value.length}B)`);
      window.dispatchEvent(new CustomEvent('blend:storage-quota-exceeded',
        { detail: { store, key, approxBytes: value.length } }));
    } else {
      console.error(`[safe-storage] setItem failed for "${key}" (store=${store})`, err);
    }
    return false;
  }
}
```
- `isQuotaError`는 `QuotaExceededError`, Firefox `NS_ERROR_DOM_QUOTA_REACHED`, code 22 / 1014 모두 인식.
- `safeSetItemWithTrim` 헬퍼도 함께 export (현재 미사용, 후속 사용 예약).

#### 2. `src/components/storage-quota-toast.tsx` (신규)
- `blend:storage-quota-exceeded` 이벤트 리스너 + 우측 하단 amber 토스트.
- ko/en 카피 자동 (`window.location.pathname.startsWith('/en')`).
- 8초 자동 dismiss, 2초 디바운스로 burst 중복 방지, 최근 3개만 유지.
- `onOpenSecurity` prop → 보안 페이지 이동 CTA (export / 모든 데이터 삭제 가능).

## 🚫 BUG-004 — API 라우트 Rate Limiting (N/A 처리)

원래 BACKLOG: "/api/web-search, /api/transcribe, /api/image-gen 등 공개 API 라우트에 rate limiting 미구현 → DDoS / API 키 남용 위험".

**조사 결과**:
- `next.config.js`에 `output:'export'` 사용 (정적 빌드, Vercel은 정적 사이트로 서빙).
- `/api/web-search/route.ts`, `/api/transcribe/route.ts`, `/api/image-gen/route.ts`, `/api/url-reader/route.ts`, `/api/webdav-proxy/route.ts`, `/api/youtube-transcript/route.ts` 전부 `export const dynamic = 'force-static'` + `{ disabled: true }` 응답만 반환하는 stub.
- 실제 OpenAI/Anthropic/Google 호출은 BYOK로 클라이언트가 provider에 직접. 서버에 rate limit 대상 자체가 존재하지 않음.

**결론**: 현재 아키텍처에서 rate limit은 의미 없음. 향후 동적 API 라우트(라이센스 검증·결제 webhook 등) 도입 시점에 IP 기반 token bucket 추가하는 것으로 보류.

BACKLOG에 ✅ 처리 + 재오픈 조건 명시.
