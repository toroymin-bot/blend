### 2차 시도 — `b543886` (09:17 KST) **[진짜 원인]**

**진단**: 1차 fix는 Date 렌더링만 잡음. 그러나 #418의 실제 원인은 **언어 의존 텍스트 + Zustand persist localStorage**.

**진짜 원인 2개**:

1. **`src/modules/ui/splash-screen.tsx`**
   - `<h1>{t('app.tagline')}</h1>` — 서버는 한국어 렌더링, 클라이언트는 URL `/en/` 라우팅 후 영어로 hydrate → text node mismatch
   - 모든 라우트에 splash가 떠서 #418이 어디서나 발생

2. **`src/modules/chat/chat-view-design1.tsx`**
   - 트라이얼 배지 `{trialRemaining}/10` — Zustand persist가 localStorage에서 값을 클라이언트 마운트 후에야 로드
   - 서버는 store 기본값 `10`, 클라이언트는 캐시된 값 (예: `7`) → hydration mismatch

**해결**: 두 위치 모두 `suppressHydrationWarning` 적용 (실제 의미: "이 텍스트는 i18n/localStorage 의존이라 클라 hydration 후 update가 정상").

**파일별 변경**:
- `src/modules/ui/splash-screen.tsx` — `<h1>` `suppressHydrationWarning`
- `src/modules/chat/chat-view-design1.tsx` — 트라이얼 배지 `<span>` `suppressHydrationWarning`

**결과**: ✅ **재테스트 필요**. 콘솔 #418 사라짐 확인 시 BUG-003 → ✅ Resolved.
